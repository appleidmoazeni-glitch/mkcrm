# MKCRM 0.9.11-cardex-paging-safe-full

پایه نسخه: `0.9.10-inventory-inline-cardex-modal`

## مشکل اصلی
در WebService شایگان، endpoint زیر برای کاردکس فقط `RowCount=1` را قبول می‌کند:

`/api/Item/GetKardex?RowStart=N&RowCount=1`

بنابراین اگر CRM فقط `RowStart=0` را بخواند، کاردکس ناقص می‌شود و برای کالاهایی مثل `P10377702113` فقط یک گردش دیده می‌شود.

## اصلاح انجام‌شده
- خواندن کاردکس به صورت صفحه‌به‌صفحه انجام می‌شود: `RowStart=0,1,2,...`
- برای جلوگیری از کندی فروشنده، سقف امن اعمال شده است:
  - فروشنده: ۲۰ صفحه/گردش
  - ادمین سریع: ۵۰ صفحه/گردش
  - ادمین کامل/مسیرهای مدیریتی: ۱۰۰ صفحه/گردش
- ردیف‌های مانده اول/آخر دوره که گردش واقعی نیستند از خروجی تحلیلی حذف می‌شوند.
- در کاردکس بازشده از موجودی، شماره انبار همان ردیف به backend ارسال می‌شود.

## تنظیمات قابل تغییر در `.env`
```env
KARDEX_SELLER_MAX_ROWS=20
KARDEX_ADMIN_QUICK_MAX_ROWS=50
KARDEX_ADMIN_FULL_MAX_ROWS=100
KARDEX_REQUEST_TIMEOUT_MS=6000
```

## تست قبل از تحویل به فروش
1. با کاربر فروشنده وارد شوید.
2. کالا `P10377702113` را در موجودی جستجو کنید.
3. از ردیف انبار موردنظر، کاردکس را باز کنید.
4. انتظار: بیشتر از یک گردش نمایش داده شود، نه فقط گردش اول.
5. با شایگان مقایسه کنید؛ اگر تعداد گردش‌ها بیشتر از سقف فروشنده باشد، پیام «نمایش سریع» دیده می‌شود.

## محدودیت آگاهانه
کاردکس کامل سنگین نباید در لحظه برای فروشنده خوانده شود؛ تحلیل خواب کالا و اتصال موجودی به خریدها/تأمین‌کننده‌ها باید در sync شبانه طراحی شود.
