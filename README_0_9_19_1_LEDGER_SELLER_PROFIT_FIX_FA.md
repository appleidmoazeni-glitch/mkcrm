# MKCRM 0.9.19.1 - Ledger Filter + Targeted Seller Profit

## اصلاحات
- جستجوی حساب در گردش حساب برای مواردی مثل «ماتریس» مقاوم‌تر شد؛ اگر SQL نتیجه خالی بدهد، کاتالوگ نرمال‌شده و جستجوی fallback استفاده می‌شود.
- در گردش حساب فقط ردیف‌هایی که واقعاً فاکتور خرید/فروش هستند clickable می‌شوند؛ شماره سندهای عادی دیگر مثل فاکتور باز نمی‌شوند.
- گزارش عملکرد فروشنده دیگر خودکار همه فروشنده‌ها را محاسبه نمی‌کند. اول فروشنده انتخاب می‌شود، سپس فقط همان فروشنده خوانده و محاسبه می‌شود تا سیستم قفل نشود.
- لیست فروشنده‌ها از userShayganMappings/users خوانده می‌شود.
- تمام اصلاحات قبلی باید حفظ شده باشد: Lead Sync، بانک مشتریان، تفکیک سایت/کنسول، نمایش فاکتور از گردش، حذف رزرو شماره، شماره نهایی لحظه صدور، HasDuplicateTrackingCode، جستجوی فارسی/عربی، فرمت سه‌رقمی.

## نصب
.env عملیاتی را overwrite نکنید. بعد نصب:

pm2 restart mkcrm --update-env

## تست
1. گردش حساب: جستجوی «ماتریس» و «ریس» را تست کنید.
2. گردش حساب: فقط فاکتورهای خرید/فروش clickable باشند، نه سندهای حقوق/انتقال/واریز.
3. عملکرد فروشنده: ابتدا فروشنده انتخاب شود، سپس گزارش محاسبه شود.
