# MKCRM 0.9.19.51 - Unified Inventory Core Refactor

این نسخه بر پایه نسخه 0.9.19.49 ساخته شده و فقط روی مسیر موجودی/کالا تمرکز دارد. Sale Snapshot، خواب تأمین‌کننده و منطق اصلی Fast Sale عمداً توسعه داده نشده‌اند.

## هدف

رفع رفتارهای رندم موجودی که به دلیل چند مسیر جدا برای Auto Sync، سرچ فاکتور، موجودی انبارها، کاردکس، پیش‌فاکتور، تابلو و refresh زنده کالا ایجاد شده بود.

## تغییرات اصلی

1. Live refresh کالا به مدل Safe Authoritative تبدیل شد.
   - حذف کور `deleteMany({ itemCode })` حذف شد.
   - پاسخ مثبت WebService برای همان کالا معتبر است و cache را اصلاح می‌کند.
   - پاسخ صفر یا خطا، موجودی سالم قبلی را کورکورانه حذف نمی‌کند.
   - انبارهایی که در پاسخ جدید نیستند با quantity=0 علامت‌گذاری می‌شوند، نه حذف فیزیکی فوری.

2. سرچ دقیق کد کالا در فروش و موجودی، همیشه refresh زنده همان کالا را trigger می‌کند.
   - اگر Mongo فقط یک انبار از چند انبار را داشته باشد، سرچ دقیق کد کالا می‌تواند سایر انبارها را اضافه کند.

3. صفحه «موجودی انبارها» از snapshot جهانی Mongo می‌خواند.
   - `/api/inventory/by-stock` دیگر برای مسیر عملیاتی از GetRemain با فیلتر انبار استفاده نمی‌کند.
   - این با تصمیم قبلی پروژه هماهنگ است که WebService در خواندن انبار به انبار ممکن است چند قلم را جا بیندازد.

4. کاردکس قبل از خواندن، context موجودی همان کالا را ensure می‌کند.
   - UI کاردکس دیگر روی «در حال خواندن کاردکس...» گیر نمی‌کند و خطای واقعی را نمایش می‌دهد.

5. قبل از صدور فاکتور فروش، validation سمت سرور اضافه شد.
   - `itemCode + stockNumber + stockGuid + quantity` کنترل می‌شود.
   - backend حق انتخاب انبار جایگزین ندارد.
   - اگر live refresh قبل از فروش صفر/مشکوک باشد، صدور فاکتور block می‌شود.

6. تبدیل پیش‌فاکتور به فاکتور نیز validation موجودی دارد.
   - در ویرایش پیش‌فاکتور، فیلدهای انبار حفظ می‌شوند: `stockNumber`, `stockName`, `stockGuid`.

7. endpoint عیب‌یابی اضافه شد:
   - `/api/inventory/debug-item?itemCode=CODE`
   - مخصوص مدیر/حسابداری/انبار/بازرگانی.

8. وضعیت Auto Sync اکنون فیلدهای نمایش ساعت تهران هم دارد.

## تست‌های پیشنهادی بعد از نصب

1. کالا `9610BP30M0` را در سرچ فاکتور با کد دقیق جستجو کن؛ باید انبارهای 21 و 69 را نشان دهد.
2. همان کالا را از صفحه موجودی انبارها و کاردکس تست کن.
3. یک کالای چندانباره را از چند انبار تست کن؛ UI نباید ردیف‌ها را collapse کند.
4. فاکتور فروش با انبار انتخابی صادر کن و پرینت شایگان را با `stockNumber` انتخابی مقایسه کن.
5. تبدیل پیش‌فاکتور به فاکتور را فقط بعد از انتخاب انبار معتبر تست کن.
