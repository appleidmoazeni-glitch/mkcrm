# MKCRM 0.9.19.27.4 - Supplier Sleep Diagnostic Fallback

## هدف
اصلاح مسیر خواب کالا تأمین‌کننده پس از شکست نسخه 0.9.19.27.3 در پیدا کردن فاکتورهای خرید تأمین‌کننده انتخاب‌شده.

## تغییرات
- اضافه شدن Diagnostic مرحله‌ای در UI و API.
- نمایش مراحل:
  1. انتخاب تأمین‌کننده
  2. خواندن گردش حساب
  3. استخراج شماره فاکتور از گردش
  4. اعتبارسنجی شماره‌ها با `getInvoice(no, 3)`
  5. fallback اسکن نزولی فاکتور خرید در صورت عدم یافتن فاکتور معتبر
- نرمال‌سازی اعداد فارسی/عربی و حذف leading zero در تطبیق کد حساب.
- تطبیق منعطف‌تر نام تأمین‌کننده در صورت اختلاف فاصله/نیم‌فاصله.
- ذخیره Diagnostic در Collection جدید `supplierSleepDiagnostics`.
- بدون SQL مستقیم.
- بدون تغییر Sale Engine.

## مسیر تست
1. مدیریت → خواب کالا تأمین‌کننده
2. آماده‌سازی
3. جستجوی تأمین‌کننده
4. انتخاب تأمین‌کننده
5. خواندن مرحله‌ای فاکتورهای خرید
6. بررسی Diagnostic مراحل
7. اگر فاکتور معتبر پیدا شد، تحلیل خواب کالای همین تأمین‌کننده

## نکته
اگر مرحله گردش حساب شماره فاکتور واقعی نداشته باشد، سیستم با fallback از آخرین فاکتور خرید به عقب اسکن می‌کند و فقط فاکتورهای منطبق با تأمین‌کننده انتخاب‌شده را نگه می‌دارد.
