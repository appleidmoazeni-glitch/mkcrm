# MKCRM 0.9.19.12 Final Seller Performance & User Cleanup

- Soft delete کاربران از صفحه کاربران اضافه شد؛ سوابق حذف نمی‌شوند.
- ایجاد کاربر جدید حفظ و فرم کاربر جدید پاک‌سازی می‌شود.
- عملکرد فروشنده به مسیر B منتقل شد: گزارش در Mongo آرشیو می‌شود و ۳۰ گزارش آخر قابل مشاهده است.
- دکمه «محاسبه دستی / بروزرسانی گزارش» گزارش جدید را از WebService فاکتور فروش + FIFO SQL محاسبه و ذخیره می‌کند.
- فیلتر نوت‌بوک با منطق کد کالا شروع‌شونده با 1 حفظ شد.
- فیلترهای سود و گروه کالا در انتهای گزارش قرار گرفتند و با بروزرسانی اعمال می‌شوند.
- ریز فاکتورها کلیک‌پذیر شد و به نمایش/چاپ فاکتور وصل می‌شود.
