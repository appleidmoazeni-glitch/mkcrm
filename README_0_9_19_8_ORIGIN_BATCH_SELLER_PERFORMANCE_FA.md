# MKCRM 0.9.19.8 - Origin Batch + Seller Performance

## هدف نسخه
این نسخه روی پایه 0.9.19.x ادامه داده شده و شاخه 0.9.20.7 مبنا نیست.

## اصلاحات اصلی
1. گزارش کل موجودی انبارهای فعال در خواب کالا با batch بندی SQL اصلاح شد تا خطای SQL Server `maximum of 2100 parameters` رخ ندهد.
2. محاسبه منشأ موجودی همچنان از SQL read-only انجام می‌شود و عملیات نوشتنی شایگان دست‌نخورده روی WebService باقی می‌ماند.
3. عملکرد فروشنده به UI مدیریتی کامل‌تر تبدیل شد: فیلتر ماه، فروشنده یا همه، فروشگاه، حداقل/حداکثر سود فاکتور، نرخ پورسانت.
4. سود فروشنده با FIFO SQL از فروش‌های ثبت‌شده CRM محاسبه می‌شود، نه آخرین قیمت ساده.
5. جدول‌های خلاصه فروشنده، ریز فاکتورها و پورسانت پیشنهادی اضافه شد.
6. دکمه قدیمی رزرو شماره فاکتور در UI حذف/غیرفعال شد.

## نکته مدیریتی
گزارش عملکرد فروشنده برای پرداخت پورسانت قابل استفاده است، اما ردیف‌های `سود نامشخص` باید قبل از پرداخت نهایی بررسی شوند.
