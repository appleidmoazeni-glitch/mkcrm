# MKCRM 0.9.16.4 - Sale Reset + Customer Bank + Logo Fix

## اصلاحات

1. بعد از صدور موفق فاکتور فروش، فرم فروش بدون نیاز به رفتن به صفحه دیگر برای فاکتور جدید آماده می‌شود:
   - ردیف‌های کالا پاک می‌شود
   - مشتری/موبایل/کدملی/Lead ID پاک می‌شود
   - شماره رزرو قبلی پاک می‌شود
   - saleIssueKey جدید ساخته می‌شود
   - دکمه صدور دوباره فعال می‌شود
   - قفل ضد ثبت تکراری برای همان فاکتور قبلی حفظ می‌شود

2. مشتری جدید ثبت‌شده در فاکتور فروش CRM در بانک مشتریان CRM upsert می‌شود:
   - اگر موبایل/کدملی/نام قبلاً وجود داشته باشد رکورد آپدیت می‌شود
   - اگر وجود نداشته باشد مشتری جدید ساخته می‌شود
   - آخرین فاکتور، آخرین خرید، فروشنده، Lead ID و مبلغ خرید ذخیره می‌شود
   - customerInvoiceHistory هم برای فاکتور CRM ثبت می‌شود

3. مشکل نمایش لوگو در سایدبار CRM اصلاح شد:
   - shell اصلی CRM لوگو را رندر می‌کند
   - دیگر فقط لحظه ورود دیده نمی‌شود و بعد از load شدن app حذف نمی‌شود

## تست سریع

1. یک فاکتور فروش با مشتری جدید و موبایل صادر کن.
2. بدون خروج از صفحه، فاکتور دوم را با مشتری/کالای جدید صادر کن.
3. در بانک مشتریان همان موبایل را سرچ کن؛ باید مشتری اضافه/آپدیت شده باشد.
4. اگر فاکتور اول را دوباره ارسال کنی، کنترل ثبت تکراری همچنان باید مانع شود.
5. بعد از لاگین، لوگوی مشهد کالا باید در سایدبار باقی بماند.
