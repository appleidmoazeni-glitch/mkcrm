# MKCRM 0.9.19.27 - Purchase Layer Supplier Sleep

## هدف
بازطراحی خواب کالا تأمین‌کننده بر اساس فاکتورهای خرید، نه کاردکس.

## تغییرات
- حذف UIهای اضافی و تستی خواب کالا/نماینده از منو.
- نگه‌داشتن فقط منوی «خواب کالا تأمین‌کننده».
- ایجاد موتور جدید `purchase-sleep`.
- خواندن فاکتورهای خرید از مسیر تست‌شده تابلو: `Invoice/Get` با `InvTyp=3` و بازه تاریخ.
- ساخت لایه‌های خرید تأمین‌کننده.
- تخصیص موجودی فعال به لایه‌های خرید با Reverse FIFO.
- تفکیک «مانده قبل سال / منشأ نامشخص».
- عدم استفاده از SQL مستقیم.
- عدم استفاده از GetKardex برای تشخیص منشأ.
- حفظ Sale Guard و مسیر صدور فاکتور نسخه‌های قبلی.

## Collectionهای جدید
- supplierPurchaseInvoices
- supplierPurchaseLayers
- supplierInventoryAllocation
- supplierSleepSummary
- supplierSleepSnapshots

## مسیر UI
مدیریت → خواب کالا تأمین‌کننده

## تست پیشنهادی
1. آماده‌سازی
2. ساخت Snapshot خرید-محور با از تاریخ 14050101
3. بررسی خلاصه تأمین‌کننده‌ها
4. بررسی مقدار مانده قبل سال / منشأ نامشخص

## هشدار
در این نسخه سود تأمین‌کننده هنوز قطعی نیست و با `profitStatus=estimated_not_final` نگهداری می‌شود. اتصال فروش به لایه خرید در فاز بعدی انجام شود.
