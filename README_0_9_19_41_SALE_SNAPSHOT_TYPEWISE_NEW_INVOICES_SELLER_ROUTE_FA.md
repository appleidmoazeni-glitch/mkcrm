# MKCRM 0.9.19.41 - Sale Snapshot Typewise New Invoices + Seller Route

## هدف
اصلاح مسیر خواندن ادامه فاکتورهای فروش و آماده‌سازی پایه عملکرد فروشنده.

## تغییرات
- خواندن Sale Snapshot به‌صورت جداگانه برای InvoiceType 6 و 7 انجام می‌شود.
- اجرای عادی فقط فاکتورهای جدید را بر اساس آخرین InvNo ذخیره‌شده هر نوع می‌خواند.
- بازسازی کامل از ابتدا با reset/full همچنان وجود دارد.
- RowStart فقط داخل همان scan استفاده می‌شود، نه به‌عنوان مبنای فاکتور جدید.
- مسیر گزارش نماینده اضافه شد: `/api/sale-snapshot/seller-performance`.
- نام نماینده کلیک‌پذیر از Mongo گزارش می‌خواند: فروش، تعداد فاکتور، شماره فاکتورها و گروه اصلی کالا.
- سود/ROI هنوز عدد قطعی تولید نمی‌کند.

## تست پیشنهادی
1. نصب و restart.
2. Sale Snapshot → آماده‌سازی.
3. اگر می‌خواهی کل داده‌ها دوباره خوانده شود، تیک «بازسازی کامل» را بزن.
4. برای اجرای روزهای بعد، بدون تیک بازسازی اجرا کن تا فقط فاکتورهای جدید خوانده شوند.
5. روی نام نماینده کلیک کن و گزارش فروش/گروه/فاکتورها را بررسی کن.
