# MKCRM 0.9.15.2 - User Admin Edit Lock

## هدف
اصلاح واقعی صفحه مدیریت کاربران: انتخاب کاربر از جدول، لود شدن اطلاعات در فرم بالا، امکان ویرایش و ذخیره نام، نام خانوادگی، نام کاربری، رمز، گروه کاربری، موبایل، وضعیت و دسترسی خرید.

## قانون جدید
اگر کاربر با اکانت خود در CRM فاکتور صادر کرده یا پیش‌فاکتور را به فاکتور تبدیل کرده باشد، اطلاعات هویتی/نام کاربری/رمز/گروه قابل ویرایش نیست. این کاربر فقط می‌تواند غیرفعال شود.

## اصلاحات
- افزودن ستون عملیات و دکمه ویرایش در لیست کاربران.
- افزودن وضعیت فعال/غیرفعال به فرم کاربران.
- نمایش قفل ویرایش برای کاربران دارای سابقه صدور فاکتور.
- قفل کردن فرم برای کاربران دارای سابقه و باز گذاشتن فقط وضعیت فعال/غیرفعال.
- کنترل سمت سرور برای جلوگیری از ویرایش کاربران دارای سابقه حتی اگر UI دور زده شود.
- حفظ اصلاحات نسخه‌های قبل: SQL Read، دوره مالی، بانک مشتریان، خواب کالا-تأمین‌کننده.
