# MKCRM 0.9.19.58 — Authoritative Live Zero Reconcile

این نسخه مشکل باقی ماندن موجودی مثبت غیرواقعی را اصلاح می‌کند.

## تغییرات
- اگر یک ردیف مثبت قدیمی در Sync کامل همان انبار دیده نشود، همان کالا با GetRemain زنده و دقیق خوانده می‌شود.
- پاسخ موفق Live برای کالا در انبارهای فعال authoritative است:
  - ردیف‌های مثبت جدید upsert می‌شوند.
  - ردیف مثبت قدیمی که در پاسخ زنده وجود ندارد، صفر می‌شود.
- جستجوی دقیق کد کالا، endpoint موجودی کالا و قبل از کاردکس نیز از reconcile زنده authoritative استفاده می‌کنند.
- پاسخ خطا یا عدم دسترسی وب‌سرویس باعث حذف موجودی نمی‌شود؛ فقط پاسخ موفق مبنای صفرکردن است.
- انبارهای غیرفعال در تصمیم موجودی فروش و reconcile دخالت داده نمی‌شوند.
- منطق تابلو نسخه 57 و صدور سریع فاکتور دست‌نخورده باقی مانده است.

## متغیر اختیاری
`INVENTORY_MISSING_LIVE_VERIFY_LIMIT=100`
حداکثر کالاهای جاافتاده از هر Sync انبار که در همان دور Live بررسی می‌شوند.
