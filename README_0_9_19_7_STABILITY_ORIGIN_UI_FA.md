# MKCRM 0.9.19.7

اصلاحات این نسخه روی پایه 0.9.19.6 انجام شده و مسیر SQL Read-Only حفظ شده است.

## تصمیم‌های اجرایی
- تفکیک کامل app.js و server.js به ماژول‌های جداگانه در این نسخه انجام نشد، چون ریسک شکستن نسخه عملیاتی بالا بود. به جای آن patch نهایی و متمرکز اضافه شد. تفکیک ماژولی باید فاز جداگانه با تست regression باشد.
- گزارش خواب کالا در UI به مدل موجودی‌محور تغییر کرد: اول کل موجودی انبارهای فعال، بعد منشا مانده کالا.

## اصلاحات
- گزارش کل موجودی انبارهای فعال در بالای صفحه خواب کالا اضافه شد.
- کلیک روی هر منشا، لایه‌های کالایی همان منشا را نمایش می‌دهد.
- منشا نامشخص / مانده از قبل با originAccountId=0 پشتیبانی شد.
- گزارش خواب کالا حتی بدون تأمین‌کننده منتخب، خلاصه کل موجودی فعال را محاسبه می‌کند.
- جدول‌ها به صورت افقی قابل اسکرول شدند.
- تاریخ کاردکس با normalizeDateDisplay یکسان‌سازی شد.
- لیست انبارها در localStorage با TTL پنج دقیقه cache می‌شود.
- صفحه تنظیمات هنگام ورود، لیست انبارهای فعال را خودکار بارگذاری می‌کند.
- صفحه نقش‌ها دیگر placeholder خالی نیست.

## مواردی که عمداً انجام نشد
- تفکیک app.js/server.js به چند ماژول در این نسخه انجام نشد؛ این کار باید فاز جدا با تست کامل باشد.
