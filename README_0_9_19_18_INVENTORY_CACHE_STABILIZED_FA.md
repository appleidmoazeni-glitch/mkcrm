# MKCRM 0.9.19.18 - WebService Inventory Cache Stabilized

این نسخه بر پایه `mkcrm_0_9_19_17_WS_FULL_READY_3FIX` ساخته شده و بخش‌های سالم نسخه قبل حفظ شده‌اند.

## اصلاحات
- هر بار موجودی live یک کالا از شایگان خوانده شود، cache موجودی Mongo همان کالا اصلاح می‌شود.
- بعد از صدور موفق فاکتور فروش، موجودی کالاهای همان فاکتور live خوانده و snapshot اصلاح می‌شود.
- اگر جستجوی فاکتور فروش در snapshot نتیجه ندهد، fallback محدود WebService اجرا می‌شود و نتیجه در Mongo upsert می‌شود.
- هنگام مشاهده موجودی یک کالا از endpoint موجودی، cache همان کالا اصلاح می‌شود.
- هنگام «اعلام ورود کالا در تابلو» از فاکتور خرید، موجودی کالاهای همان فاکتور خرید در cache اصلاح می‌شود.
- `syncCatalog` دیگر در sync ناقص یا خروجی صفر، cache قبلی را پاک نمی‌کند.
- `.env` عملیاتی WebService-only داخل فایل قرار گرفت؛ بدون تنظیمات SQL.

## نکته نصب
قبل از جایگزینی، از `.env`، MongoDB و پوشه CRM فعلی بکاپ بگیرید.
