# MKCRM 0.9.19.33 - Supplier Profit Diagnostic + Snapshot Label

تمرکز نسخه:

- Diagnostic کامل مسیر سود تأمین‌کننده: فروش‌های خوانده‌شده، Body فاکتور فروش، ردیف‌های فروش هدف، لایه‌های خرید جهانی، تخصیص به LayerId و دلایل عدم Match.
- اصلاح لیست Snapshotهای اخیر: نمایش نام تأمین‌کننده + تاریخ شمسی گزارش، در کنار کد Snapshot.
- حفظ اصلاح موجودی انبار و dedupe نسخه 0.9.19.32.
- حفظ True Reverse FIFO، LayerId، Validation/Reconciliation و فیلتر گروه اصلی.

## تست پیشنهادی

1. خواب کالا تأمین‌کننده را برای همان تأمین‌کننده قبلی اجرا کن.
2. بخش Diagnostic سود را بررسی کن:
   - فاکتور فروش اسکن‌شده
   - Body فروش خوانده‌شده
   - ردیف فروش مرتبط با کالاهای هدف
   - تخصیص سود به LayerId منتخب
   - Hit روی لایه تأمین‌کننده انتخابی
3. اگر سود هنوز صفر بود، خروجی Diagnostic مشخص می‌کند مشکل از خواندن فروش است یا از Match به LayerId.
4. Snapshotهای اخیر باید با نام تأمین‌کننده و تاریخ شمسی دیده شوند، نه فقط کد خام.
