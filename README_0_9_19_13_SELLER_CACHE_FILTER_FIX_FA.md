# MKCRM 0.9.19.13

اصلاح عملکرد فروشنده:
- فیلتر سود و فیلتر نوت‌بوک بعد از تولید گزارش فقط روی کش آخرین گزارش اعمال می‌شوند.
- تغییر فیلترهای تکمیلی دیگر WebService و FIFO را دوباره اجرا نمی‌کند.
- فیلتر نوت‌بوک بر اساس شروع کد کالا با رقم 1 اعمال می‌شود.
- برای فاکتورهای ترکیبی، در صورت وجود lines ذخیره‌شده، فقط ردیف‌های نوت‌بوک در جمع فروش/سود فیلترشده وارد می‌شوند.
- کلید آرشیو sellerPerformanceHistory فقط فیلترهای پایه را شامل می‌شود: تاریخ، فروشنده، فروشگاه، نرخ پورسانت و جریمه Lead.
