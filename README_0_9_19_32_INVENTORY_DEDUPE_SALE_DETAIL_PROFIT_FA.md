# MKCRM 0.9.19.32 — Inventory Dedupe + Sale Detail Profit Fix

## هدف نسخه
این نسخه فقط دو مسیر را اصلاح می‌کند:

1. خروجی مانده انبار CRM باید با خروجی کامل WebService شایگان یکی شود.
2. سود تأمین‌کننده نباید به دلیل header-only بودن خروجی فاکتور فروش، همیشه `NO_MATCHED_SALE_OR_UNSOLD` بماند.

## اصلاح 1 — خروجی مانده انبار / انبارگردانی
در مسیرهای موجودی، قبل از نمایش و export، ردیف‌ها با کلید زیر یکتا می‌شوند:

```text
StoreNumber + ItemCode
```

اگر یک ردیف دقیقاً تکراری باشد، فقط یک‌بار نمایش داده می‌شود. اگر تکراری با مقدار متفاوت باشد، حذف کور انجام نمی‌شود و conflict در خروجی diagnostic برمی‌گردد.

فیلدهای diagnostic اضافه‌شده:

```text
rawCount
uniqueCount
duplicateCount
duplicateConflicts
```

## اصلاح 2 — سود تأمین‌کننده
در نسخه قبلی، خواندن فروش از `Invoice/Get` تاریخ‌محور ممکن بود فقط header فاکتور را برگرداند و Body نداشته باشد. در این حالت هیچ ردیف فروش پیدا نمی‌شد و سود روی:

```text
NO_MATCHED_SALE_OR_UNSOLD
```

می‌ماند.

در این نسخه اگر Body فاکتور فروش خالی باشد، سیستم با شماره همان فاکتور:

```text
getInvoice(no, 2)
```

را اجرا می‌کند و Body کامل را می‌خواند. سپس فروش‌ها به LayerIdهای خرید با FIFO وصل می‌شوند.

## بدون تغییر
- Sale Engine تغییر نکرده است.
- SQL مستقیم استفاده نشده است.
- True Reverse FIFO موجودی تأمین‌کننده حفظ شده است.
- LayerId حفظ شده است.
- فیلتر گروه اصلی حفظ شده است.

1. تأمین‌کننده‌ای را انتخاب کن که فروش واقعی از خریدهایش وجود دارد.
2. `حداکثر صفحات فروش` را اول روی 600 بگذار.
3. تحلیل را اجرا کن.
4. در diagnostic فروش بررسی کن:

```text

```
