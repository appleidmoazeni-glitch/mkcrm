# MKCRM 0.9.19.27.3 - Supplier Ledger Invoice Sleep

## هدف
اصلاح مسیر خواب کالا تأمین‌کننده انتخاب‌شده بر اساس گردش حساب تأمین‌کننده و استخراج شماره فاکتورهای خرید.

## اصلاحات
- رفع خطای `readBody is not defined` در APIهای تأمین‌کننده انتخاب‌شده.
- اتصال routeها به `collectBody` و `connectMongo` مطابق ساختار پروژه.
- استخراج گسترده‌تر شماره فاکتور خرید از ردیف‌های گردش حساب تأمین‌کننده.
- استفاده از `getInvoice(no, 3)` فقط برای شماره‌های استخراج‌شده از گردش همان تأمین‌کننده.
- اعتبارسنجی فاکتور با `sameSupplier` تا فاکتور اشتباه وارد تحلیل نشود.
- حفظ Sale Guard و عدم تغییر مسیر صدور فاکتور.

## مسیر تست
1. مدیریت → خواب کالا تأمین‌کننده
2. آماده‌سازی
3. جستجوی تأمین‌کننده
4. انتخاب تأمین‌کننده
5. خواندن فاکتورهای خرید همان تأمین‌کننده
6. ساخت/تحلیل Snapshot همان تأمین‌کننده

## محدودیت فعلی
محاسبه مانده فعلی هنوز از لایه‌های خرید تأمین‌کننده انتخاب‌شده و موجودی فعلی استفاده می‌کند. تحلیل دقیق سود و مانده هر فاکتور با کاردکس اقلام همان فاکتور باید در فاز بعدی تکمیل شود.
