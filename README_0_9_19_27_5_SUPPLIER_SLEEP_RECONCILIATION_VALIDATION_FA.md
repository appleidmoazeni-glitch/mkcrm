# MKCRM 0.9.19.27.5 - Supplier Sleep Reconciliation Validation

## هدف
اصلاح موتور خواب کالا تأمین‌کننده بر پایه کنترل صحت فاکتور خرید و لایه کالایی.

## تغییرات اصلی
- خرید دوره حسابداری از جمع فاکتورهای خرید معتبر تأمین‌کننده محاسبه می‌شود.
- خرید کالایی تحلیل‌شده از جمع لایه‌های کالایی Body فاکتور محاسبه می‌شود.
- اختلاف فاکتور و لایه برای هر فاکتور و کل تأمین‌کننده محاسبه و در UI نمایش داده می‌شود.
- وضعیت اعتبارسنجی اضافه شد:
  - OK
  - MINOR_ADJUSTMENT
  - WARN_LAYER_LESS_THAN_INVOICE
  - INFO_LAYER_MORE_THAN_INVOICE
  - FAIL_NO_LAYERS
  - BODY_MISSING_OR_NOT_PARSED
  - PARTIAL_LINES_PARSED_RISK
  - UNDER_PARSED_RISK_OR_NON_ITEM_CHARGE
  - COST_ALLOCATED_OVER_OR_LANDED_COST
- Diagnostic قبلی حفظ شد.
- Sale Engine تغییر نکرده است.
- SQL مستقیم استفاده نشده است.

## قانون مدیریتی
- خرید دوره = جمع فاکتورهای معتبر
- مانده، رسوب، سن و لایه‌های موجودی = بر اساس لایه‌های کالا
- اگر جمع لایه‌ها کمتر از جمع فاکتورها باشد، سیستم هشدار ریسک تحلیل ناقص یا هزینه غیرکالایی می‌دهد.

## تست پیشنهادی
1. مدیریت → خواب کالا تأمین‌کننده
2. آماده‌سازی
3. جستجوی تأمین‌کننده
4. خواندن مرحله‌ای فاکتورهای خرید
5. تحلیل و کنترل صحت
6. بررسی جدول اختلاف فاکتور/لایه
