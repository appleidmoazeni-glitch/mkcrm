# MKCRM 0.9.19.27.1 - Purchase Layer Supplier Index Sleep

## هدف
اصلاح مسیر خواب کالا تأمین‌کننده بر پایه فاکتورهای خرید، نه کاردکس‌محور.

## تغییرات
- UI فقط «خواب کالا تأمین‌کننده» را نشان می‌دهد؛ UIهای اضافی خواب کالا حذف/redirect شدند.
- مرحله اول: خواندن فاکتورهای خرید سال جاری و لیست تأمین‌کنندگان دارای خرید.
- نمایش تعداد فاکتور، مبلغ خرید دوره، تعداد SKU و ردیف کالا برای هر تأمین‌کننده.
- نمایش فاکتورهای خرید هر تأمین‌کننده با کلیک روی دکمه «فاکتورها».
- Snapshot تحلیلی همچنان با Purchase Layers و Reverse FIFO ساخته می‌شود.
- شاخص رسوب از ۹۰/۱۸۰ روز به بیش از ۳۰ روز و بیش از ۶۰ روز تغییر کرد.
- اگر مسیر تاریخ‌محور Invoice/Get در شایگان خروجی ندهد، fallback تست‌شده تابلو استفاده می‌شود: آخرین شماره فاکتور خرید + خواندن نزولی `getInvoice(no, 3)`.
- SQL مستقیم استفاده نشده است.
- Sale Guard نسخه‌های قبلی حفظ شده است.

## مسیر تست
1. مدیریت → خواب کالا تأمین‌کننده
2. آماده‌سازی
3. خواندن فاکتور خرید و لیست تأمین‌کنندگان
4. انتخاب یک تأمین‌کننده و بررسی فاکتورها
5. ساخت Snapshot تحلیلی
6. بررسی خلاصه خواب کالا و شاخص‌های رسوب +۳۰ و +۶۰ روز

## نکته
در این نسخه هدف اول، تأیید خواندن فاکتورهای خرید و ساخت ایندکس تأمین‌کننده است. اگر تعداد فاکتور خرید صفر شد، باید خروجی fallback و max scan بررسی شود.
