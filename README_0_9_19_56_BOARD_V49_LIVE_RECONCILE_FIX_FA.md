# MKCRM 0.9.19.56 — اصلاح تابلو با بازگردانی منطق سالم نسخه 49

## هدف

بازگرداندن مسیر عملیاتی تابلو اتمام کالا که تا نسخه 49 بر پایه بررسی زنده و هدفمند موجودی همان کالا کار می‌کرد، بدون حذف اصلاحات سالم نسخه‌های 53 تا 55 شامل کسر فوری موجودی محلی بعد از فروش.

## اصلاح اصلی

پس از صدور موفق فاکتور فروش:

1. موجودی محلی همان `itemCode + stockNumber` فوراً کم می‌شود.
2. تابع تابلو مانند نسخه 49 برای همان کالا `refreshInventoryCacheForItem` را به‌صورت هدفمند اجرا می‌کند.
3. موجودی فقط در انبارهای فعال محاسبه می‌شود.
4. مقدار `afterQty` ثبت‌شده از فروش موفق برای انبار فروخته‌شده قطعی در نظر گرفته می‌شود تا پاسخ دیرهنگام شایگان نتواند آخرین موجودی فروخته‌شده را دوباره مثبت کند.
5. اگر جمع موجودی انبارهای فعال صفر باشد، رویداد `stock_out` ساخته می‌شود.
6. تصمیم کامل در `sale_issue_post_board_stock_out` ثبت می‌شود.

## محافظت‌های حفظ‌شده

- صدور سریع فاکتور با `InvNo=0`
- اجرای عملیات پس از فروش در background
- کسر فوری موجودی محلی
- مسیر تبدیل پیش‌فاکتور به فاکتور
- Active Warehouses
- Auto Sync و Live Repair Search
- Sale Snapshot و Supplier Sleep
- جلوگیری از ساخت رویداد تکراری باز برای همان کالا

## تست بعد از نصب

بعد از فروش آخرین موجودی یک کالا:

- `sale_issue_local_inventory_deduct` باید `afterQty: 0` داشته باشد.
- `sale_issue_post_board_stock_out` باید `totalQtyAfter: 0` و `created: true` یا در صورت وجود رویداد قبلی `reason: open-event-exists` داشته باشد.
- `/api/board/events?type=stock_out&limit=50` باید کد کالا و شماره فاکتور را نشان دهد.
