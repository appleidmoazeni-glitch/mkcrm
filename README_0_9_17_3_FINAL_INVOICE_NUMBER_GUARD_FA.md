# MKCRM 0.9.17.3 - Final Invoice Number Guard

اصلاحات:
- حذف دکمه رزرو شماره از UI فاکتور فروش.
- شماره قطعی فاکتور فقط در لحظه صدور از شایگان خوانده و کنترل می‌شود.
- قبل از insert، وجود شماره در شایگان چک می‌شود.
- در خطای duplicate key، سیستم تا ۳ بار با شماره جدید retry می‌کند.
- پارامتر HasDuplicateTrackingCode=false در ارسال فاکتور به شایگان اضافه شد.
- تنظیمات Lead ID در .env.example حفظ شده است. هنگام نصب، .env عملیاتی سرور را overwrite نکنید.
