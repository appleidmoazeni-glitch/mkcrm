# MKCRM 0.9.19.39 - Sale Snapshot Incremental Seller Map

## هدف
این نسخه Sale Snapshot را از حالت اسکن تکراری خارج می‌کند و به حالت ادامه‌پذیر/Incremental می‌برد.

## تغییرات اصلی
- خواندن فروش با `Invoice/Get` و `InvoiceType 6 تا 7` حفظ شد.
- `RowCount=20` و `RowStart` مرحله‌ای حفظ شد.
- در اجرای عادی، سیستم فقط از `nextRowStart` قبلی ادامه می‌دهد و فاکتورهای جدید را می‌خواند.
- اگر Snapshot قبلی از نسخه 0.9.19.38 وجود داشته باشد ولی state نداشته باشد، شروع بعدی از تعداد فاکتورهای خوانده‌شده قبلی تخمین زده می‌شود.
- امکان بازسازی کامل از ابتدا فقط با تیک `بازسازی کامل از ابتدا` در UI فعال می‌شود.
- اتصال فروش به نماینده واقعی با Mapping صفحه «تعریف صندوق/نماینده» انجام می‌شود.
- ستون Issuer از گزارش مدیریتی حذف شد؛ Issuer فقط در داده خام/Diagnostic باقی می‌ماند.
- سود تأمین‌کننده همچنان عمداً توسعه داده نشده است.

## تست پیشنهادی
1. نصب نسخه و تأیید نمایش `0.9.19.39-sale-snapshot-incremental-seller-map` در UI.
2. رفتن به `مدیریت → Sale Snapshot`.
3. اجرای عادی بدون تیک بازسازی کامل.
4. بررسی:
   - `RowStart` باید از نقطه قبلی ادامه دهد.
   - اگر فاکتور جدیدی نیست، تعداد جدید کم یا صفر خواهد بود.
   - نماینده فروش باید به‌جای Issuer نمایش داده شود.
5. برای تست کامل فقط یک بار با تیک بازسازی کامل اجرا شود.
