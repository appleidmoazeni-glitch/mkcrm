# MKCRM 0.9.14.0 - SQL Read Fast Full

این نسخه بر پایه نسخه کامل 0.9.13.4 ساخته شده و پچ جداگانه نیست.

## تغییر معماری

خواندن‌های سنگین از وب‌سرویس شایگان خارج شده و به SQL مستقیم فقط خواندنی منتقل شده است:

- سرچ فاکتور فروش: SQL مستقیم، فقط کالا/انبارهای دارای موجودی
- منوی موجودی انبارها: SQL مستقیم، فقط موجودی مثبت
- سرچ کالا: SQL مستقیم
- کاردکس کالا: SQL مستقیم از AC_4001_N / AC_4101_N
- گردش حساب: SQL مستقیم از AC_1001_N / AC_1101_N / AC_0101_N

ثبت فاکتور فروش، خرید و عملیات نوشتنی همچنان از WebService شایگان انجام می‌شود.

## تنظیمات SQL

در فایل .env:

```env
SHAYGAN_SQL_ENABLED=true
SHAYGAN_SQL_SERVER=192.168.1.253
SHAYGAN_SQL_PORT=1433
SHAYGAN_SQL_DATABASE=CY000001
SHAYGAN_SQL_USER=sa
SHAYGAN_SQL_PASSWORD=server
```

پیشنهاد عملیاتی: به جای sa یک user فقط خواندنی برای دیتابیس شایگان تعریف شود.

## تست بعد از نصب

1. سلامت برنامه:

```bat
curl http://127.0.0.1:1385/api/version
```

2. سلامت SQL:

```bat
curl http://127.0.0.1:1385/api/shaygan/sql-health
```

3. تست سرچ سریع فروش:

در فاکتور فروش عبارت E1504 را جستجو کنید. باید کالاهای دارای موجودی و انبارها سریع نمایش داده شوند.

4. تست موجودی انبارها:

در منوی موجودی انبارها، نتیجه فقط کالاهای دارای موجودی باشد.

5. تست کاردکس:

کالای P10377702113 را تست کنید. کاردکس باید SQL-based و بدون محدودیت RowCount=1 وب‌سرویس نمایش داده شود.

6. تست گردش حساب:

از منوی گردش حساب، یک حساب را انتخاب و گردش را بگیرید. خروجی باید از SQL مستقیم سریع‌تر برگردد.

## Rollback

اگر خطای SQL یا dependency داشتید:

1. در .env مقدار زیر را بگذارید:

```env
SHAYGAN_SQL_ENABLED=false
```

2. برنامه را restart کنید. سیستم به مسیر WebService / snapshot قبلی fallback می‌کند.

## نکته امنیتی

این نسخه هیچ INSERT/UPDATE/DELETE مستقیم روی SQL شایگان انجام نمی‌دهد. فقط SELECT دارد.
