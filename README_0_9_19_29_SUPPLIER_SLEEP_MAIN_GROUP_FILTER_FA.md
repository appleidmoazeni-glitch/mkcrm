# MKCRM 0.9.19.29 - Supplier Sleep Main Group Filter

تمرکز این نسخه فقط اصلاح گزارش گروه اصلی کالا در خواب کالا تأمین‌کننده است.

## تغییرات

- روش تشخیص گروه اصلی کالا بررسی و اصلاح شد.
- برای هر Purchase Layer فیلدهای زیر ذخیره می‌شود:
  - `mainGroupCode`
  - `mainGroupName`
  - `mainGroup`
  - `mainGroupSource`
- اولویت تشخیص گروه اصلی:
  1. فیلدهای گروه اصلی داخل Body فاکتور خرید، اگر WebService برگرداند.
  2. `itemInventoryCatalog` و raw موجودی که معمولاً شامل `ItemMainGroupCode` است.
  3. `itemCatalogAll` و `itemCatalog`.
  4. fallback بر اساس پیشوند کد کالا؛ برای نمونه کد اصلی `1` به Notebook / نوت‌بوک نگاشت می‌شود.
- گزارش گروه کالایی به گزارش **گروه اصلی کالا** محدود شد.
- زیرگروه عمداً وارد این فاز نشده است.
- در UI فیلتر گروه اصلی اضافه شد.
- Sale Engine، صدور فاکتور، Inventory Search، Lead و Tablo تغییر نکرده‌اند.
- SQL مستقیم استفاده نشده است.

## تست پیشنهادی

1. مدیریت → خواب کالا تأمین‌کننده
2. انتخاب تأمین‌کننده
3. خواندن مرحله‌ای فاکتورهای خرید
4. تحلیل و کنترل صحت
5. بررسی بخش «گزارش گروه اصلی کالا»
6. انتخاب گروه اصلی مثل `1 - Notebook / نوت‌بوک`
7. کنترل اینکه مانده، خرید، رسوب +۳۰ و +۶۰ فقط برای همان گروه نمایش داده شود.
