# MKCRM 0.9.19.31 - Supplier Sleep Profit / ROI by LayerId

## هدف نسخه
این نسخه روی آخرین نسخه عملیاتی خواب کالای تأمین‌کننده ساخته شده و سود/ROI مدیریتی را به موتور Supplier Sleep اضافه می‌کند.

## اصول حفظ‌شده
- Sale Engine تغییر نکرده است.
- SQL مستقیم استفاده نشده است.
- True Reverse FIFO نسخه 0.9.19.30 حفظ شده است.
- LayerId پایدار حفظ شده است.
- Validation و Reconciliation حفظ شده است.
- گروه اصلی کالا فقط به‌عنوان فیلتر مدیریتی حفظ شده و زیرگروه اضافه نشده است.

## منطق سود
1. فاکتورهای خرید تأمین‌کننده انتخاب‌شده خوانده می‌شود.
2. برای کالاهای همان تأمین‌کننده، خریدهای همان کالا از همه تأمین‌کنندگان خوانده می‌شود.
3. موجودی فعلی با True Reverse FIFO به آخرین خریدهای واقعی وصل می‌شود.
4. فاکتورهای فروش همان کالاها از بازه انتخابی خوانده می‌شود.
5. فروش‌ها با FIFO مدیریتی روی Purchase Layerها تخصیص داده می‌شوند.
6. اگر فروش به LayerId تأمین‌کننده انتخاب‌شده برسد، سود محاسبه می‌شود:

فروش تخصیص‌یافته - بهای خرید همان LayerId = سود FIFO مدیریتی

## هشدار مدیریتی
سود این نسخه سود قطعی حسابداری نیست. این عدد برای تحلیل مدیریتی تأمین‌کننده، ROI، رسوب و تصمیم خرید استفاده می‌شود و با وضعیت confidence ذخیره می‌شود.

## تست پیشنهادی
1. تأمین‌کننده‌ای را انتخاب کن که هم خرید و هم فروش واقعی داشته باشد.
2. Snapshot بساز.
3. داشبورد مدیریتی تأمین‌کننده را بررسی کن.
4. مانده واقعی هر فاکتور را بررسی کن.
5. لایه‌ها را باز کن و فروش مرتبط، سود FIFO و ROI را بررسی کن.
6. برای کالای 9710R50700 دوباره بررسی کن که مانده به رایان کاوه برنگردد اگر خرید آخر از مانسانا بوده است.
