# MKCRM 0.9.14.1 - انتخاب دوره مالی SQL شایگان

این نسخه برای مشکل چند دوره مالی شایگان اصلاح شده است.

## منطق جدید
- CRM هنگام بالا آمدن دیتابیس‌های `CY%` را از SQL Server می‌خواند.
- حالت پیش‌فرض: `auto` یعنی آخرین دیتابیس CY به عنوان دوره فعال انتخاب می‌شود.
- ادمین می‌تواند از منوی «تنظیمات» حالت دستی انتخاب کند.
- خواندن مستقیم SQL برای موجودی، کاردکس و گردش حساب از دیتابیس فعال انجام می‌شود.
- ثبت فاکتور و عملیات نوشتنی همچنان از WebService شایگان انجام می‌شود.

## تست بعد از نصب
```bat
curl http://127.0.0.1:1385/api/shaygan/sql-health
```
باید در خروجی `database` برابر دوره مالی فعال، مثلا `CY000002` باشد.

## مسیر UI
ورود با ادمین:

`مدیریت > تنظیمات > تنظیمات دوره مالی شایگان`

حالت پیشنهادی: «خودکار - آخرین دیتابیس CY».

## نکته عملیاتی
اگر سال مالی جدید ایجاد شد، در حالت خودکار CRM بعد از restart آخرین دیتابیس CY را انتخاب می‌کند. اگر حالت دستی فعال باشد، مدیر باید دوره جدید را انتخاب کند.
