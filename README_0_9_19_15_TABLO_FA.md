# MKCRM 0.9.19.15 - تابلو داخلی CRM

## هدف
افزودن بخش «تابلو» داخل CRM برای دو رویداد عملیاتی:

- اتمام کالا: بعد از صدور فاکتور فروش، اگر موجودی کل همان کالا در تمام انبارهای فعال صفر شود، CRM خودکار رویداد تابلو ایجاد می‌کند.
- ورود کالا: کاربر مجاز ۳۰ فاکتور خرید آخر شایگان را از WebService مرور می‌کند؛ اگر فاکتور خرید نیاز به اعلام داشته باشد، با دکمه «اعلام ورود در تابلو» ردیف‌های همان فاکتور وارد تابلو می‌شوند.

## منبع داده
- موجودی اتمام کالا: SQL read-only با کاربر امن mkcrm_reader و فقط روی انبارهای فعال.
- مرور فاکتورهای خرید: WebService رسمی شایگان، بدون write به SQL.
- ثبت رویداد تابلو: MongoDB داخلی CRM.

## دسترسی‌ها
در صفحه کاربران سه دسترسی جدید اضافه شد:

- مرور فاکتورهای خرید شایگان
- اعلام ورود کالا در تابلو
- مدیریت وضعیت‌های تابلو

نقش‌های admin / warehouse / purchase / accounting به صورت پیش‌فرض دسترسی مدیریتی/مرور دارند.

## وضعیت‌های تابلو
- جدید
- بررسی شد
- سایت بروزرسانی شد
- بسته شد

## ریسک کنترل‌شده
اتصال مستقیم به گروه تلگرام فعلاً اضافه نشده است. اعلام ورود کالا به صورت دستی و کنترل‌شده از روی فاکتور خرید انجام می‌شود تا اصلاح موجودی، انتقال داخلی، برگشت کالا یا خرید غیرقابل فروش اشتباهاً وارد تابلو نشود.
