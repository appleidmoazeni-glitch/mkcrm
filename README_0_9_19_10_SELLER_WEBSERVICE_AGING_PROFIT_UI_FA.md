# MKCRM 0.9.19.10

اصلاحات:
- عملکرد فروشنده از فاکتورهای فروش WebService شایگان خوانده می‌شود، نه فقط لاگ CRM.
- فروشنده از نماینده/جاری کارکنان فاکتور فروش (SAccountNumber) استخراج می‌شود، نه صندوق.
- فهرست فروشنده‌ها از mapping نماینده/جاری کارکنان ساخته می‌شود.
- فیلتر سود بالای X و سود زیر Y روی سود FIFO هر فاکتور اعمال می‌شود.
- جدول خواب کالا/تأمین‌کننده دوباره مبلغ سود FIFO و درصد سود FIFO را در خلاصه منشا و خلاصه تأمین‌کننده نمایش می‌دهد.
- محاسبه سود همچنان با SQL Read-Only و FIFO انجام می‌شود؛ عملیات نوشتنی با WebService است.
