# MKCRM 0.9.19.30 - Supplier Sleep True Reverse FIFO

## هدف
اصلاح مانده کالا در خواب کالا تأمین‌کننده برای کالاهایی که از یک تأمین‌کننده خرید و فروخته شده‌اند و سپس همان کالا از تأمین‌کننده دیگری مجدداً خریداری شده است.

## مشکل نسخه قبل
در نسخه 0.9.19.29 تخصیص مانده فقط روی لایه‌های خرید همان تأمین‌کننده انتخاب‌شده انجام می‌شد. در نتیجه اگر کالا قبلاً از تأمین‌کننده A خرید و فروخته شده بود و بعداً همان کالا از تأمین‌کننده B خریداری شده و موجود بود، ممکن بود مانده به اشتباه برای تأمین‌کننده A نمایش داده شود.

## اصلاح نسخه 0.9.19.30
- خرید دوره و Validation تأمین‌کننده انتخاب‌شده حفظ شده است.
- برای محاسبه مانده، سیستم برای ItemCodeهای همان تأمین‌کننده، خریدهای همان کالا را از همه تأمین‌کنندگان به صورت هدفمند و نزولی می‌خواند.
- موجودی فعلی هر کالا با TRUE Reverse FIFO از آخرین خریدهای همه تأمین‌کنندگان تخصیص داده می‌شود.
- سپس فقط لایه‌هایی که واقعاً به تأمین‌کننده انتخاب‌شده تعلق دارند در گزارش همان تأمین‌کننده نمایش داده می‌شوند.

## اثر مورد انتظار
اگر کالا `9710R50700` قبلاً از رایان کاوه خرید و فروخته شده و بعداً از مانسانا خرید شده و موجود است، مانده نباید دیگر برای رایان کاوه بیاید؛ باید برای تأمین‌کننده آخر که موجودی واقعی از آن باقی مانده نمایش داده شود.

## تغییرات حفظ‌شده
- Validation/Reconciliation
- LayerId پایدار
- مانده واقعی هر فاکتور خرید
- گزارش گروه اصلی کالا
- فیلتر گروه اصلی
- Sale Engine بدون تغییر
- بدون SQL مستقیم

## تست پیشنهادی
1. نصب نسخه
2. ورود به مدیریت → خواب کالا تأمین‌کننده
3. انتخاب رایان کاوه
4. اجرای تحلیل
5. بررسی کد کالا `9710R50700`
6. انتظار: مانده این کالا برای رایان کاوه صفر یا حذف‌شده باشد، اگر موجودی فعلی از خرید مانسانا باقی مانده است.
