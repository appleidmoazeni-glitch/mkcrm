# MKCRM 0.9.8

این نسخه برای جلوگیری از ثبت تکراری فاکتور فروش ساخته شده است.

تست اصلی:
1. فاکتور فروش جدید بساز.
2. بعد از زدن «صدور فاکتور»، چند بار سریع روی دکمه کلیک کن.
3. باید فقط یک درخواست واقعی به شایگان برود.
4. اگر درخواست دوم برسد، backend با قفل saleIssueLocks آن را reject/duplicate می‌کند.

ماژول‌های Freeze دست‌کاری نشده‌اند: گردش حساب، خرید، پیش‌فاکتور، کاردکس، موجودی، بانک مشتریان.
