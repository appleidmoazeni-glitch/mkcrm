# MKCRM 0.9.19.54 - اصلاح تبدیل پیش‌فاکتور، کسر موجودی و تابلو اتمام کالا

این نسخه بر پایه 0.9.19.53 ساخته شده و یک اصلاح محدود و عملیاتی دارد.

## موارد حفظ‌شده

- سرعت صدور فاکتور فروش عادی حفظ شده است: `InvNo=0` و شماره‌گذاری خودکار شایگان.
- عملیات بعد از فروش عادی همچنان در پس‌زمینه انجام می‌شود.
- اصلاحات موجودی نسخه 53، sync انبارهای فعال، live check بعد از انتخاب کالا، local sale deduct و تابلو فروش عادی حفظ شده‌اند.
- Sale Snapshot و Supplier Sleep تغییر نکرده‌اند.

## اصلاح جدید

در مسیر تبدیل پیش‌فاکتور به فاکتور، بعد از موفقیت `putSaleInvoice`:

1. موجودی محلی CRM برای `itemCode + stockNumber` کم می‌شود.
2. سپس تابلو اتمام کالا از snapshot بعد از کسر موجودی تصمیم می‌گیرد.
3. اگر فروش، آخرین عدد در همه انبارهای فعال باشد، رویداد اتمام کالا ثبت می‌شود.
4. لاگ `proforma_convert_local_inventory_deduct` برای عیب‌یابی ثبت می‌شود.

## تست پیشنهادی

1. یک پیش‌فاکتور برای کالایی بسازید که آخرین موجودی در همه انبارهای فعال است.
2. آن را به فاکتور تبدیل کنید.
3. موجودی همان کالا باید در CRM صفر شود.
4. تابلو باید اتمام کالا را ثبت کند.
5. فاکتور فروش عادی نیز مانند نسخه 53 تست شود تا سرعت و post-processing حفظ‌شده تأیید شود.
